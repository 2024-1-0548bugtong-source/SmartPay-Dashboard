# SmartPay Complete Flow Diagram\n\n## Hardware Interaction Flow\n\n```\nв”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ\nв”Ӯ                          SmartPay Checkout Flow                         в”Ӯ\nв””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ\n\n1. STATE_READY\n   в”ңв”Җ LCD: \"SmartPay Ready\"\n   в”ңв”Җ Serial: \"SmartPay Ready\"\n   в”ңв”Җ PIR: Listening for motion\n   в””в”Җ вҶ’ Next: Wait for customer\n\n2. CUSTOMER DETECTED (PIR triggered)\n   в”ңв”Җ Increment entry counter\n   в”ңв”Җ Serial: \"Entry: <number>\"\n   в”ңв”Җ Serial: \"Customer Entered\"\n   в”ңв”Җ LCD: \"Customer Entered\"\n   в”ңв”Җ Buzzer: 1x beep\n   в””в”Җ вҶ’ Next: Wait for product on scale\n\n3. PRODUCT ON SCALE (3kg load cell reads weight)\n   в”ңв”Җ Check weight against thresholds\n   в”ңв”Җ Product One: 150-250g вҶ’ Serial: \"Product Removed. Pay Product One (PHP5).\"\n   в”ңв”Җ Product Two: 50-120g вҶ’ Serial: \"Product Removed. Pay Product Two (PHP10).\"\n   в”ңв”Җ LCD: Show product name and price\n   в”ңв”Җ Buzzer: 2x beeps\n   в””в”Җ вҶ’ Next: Wait for coins on 1kg scale\n\n4. PRODUCT REMOVED FROM SCALE\n   в”ңв”Җ Serial: \"Pay Product One (PHP5)\" or \"Pay Product Two (PHP10)\"\n   в”ңв”Җ LCD: \"Pay <Product Name>\"\n   в”ңв”Җ Start listening to 1kg coin scale\n   в””в”Җ вҶ’ Next: Await payment deposit\n\n5. COINS DETECTED (1kg load cell reads вүҘ0.5g)\n   в”ңв”Җ Wait 1 second for settling\n   в”ңв”Җ Read final coin weight\n   в”ңв”Җ Serial: \"Coins: <weight>g - <STATUS>\"\n   в”Ӯ\n   в”ңв”Җ IF Product One (PHP5):\n   в”Ӯ  в””в”Җ Need вүҘ5.0g to verify\n   в”Ӯ\n   в”ңв”Җ IF Product Two (PHP10):\n   в”Ӯ  в””в”Җ Need вүҘ10.0g to verify\n   в”Ӯ\n   в””в”Җ вҶ’ Next: Either PAYMENT_OK or PAYMENT_FAILED\n\n6a. PAYMENT OK\n    в”ңв”Җ Serial: \"Payment OK\"\n    в”ңв”Җ LCD: \"Payment OK\\nThank you!\"\n    в”ңв”Җ Buzzer: 2x long beeps (200ms each)\n    в”ңв”Җ Wait 3 seconds\n    в””в”Җ вҶ’ Next: Back to READY\n\n6b. PAYMENT FAILED (insufficient coins)\n    в”ңв”Җ Serial: \"Add More Coins\"\n    в”ңв”Җ LCD: \"Add More\\nCoins\"\n    в”ңв”Җ Buzzer: 1x beep\n    в”ңв”Җ Wait for more coins (5 second retry window)\n    в”Ӯ  в”ңв”Җ IF more coins: Add weight and check again (go to step 5)\n    в”Ӯ  в”ңв”Җ ELSE timeout: Reset\n    в””в”Җ вҶ’ Next: Back to READY\n\n7. CLEANUP & RESET\n   в”ңв”Җ Serial: \"Customer Left\"\n   в”ңв”Җ Serial: \"SmartPay Ready\"\n   в”ңв”Җ LCD: \"SmartPay Ready\"\n   в”ңв”Җ Tare both scales (reset to 0g)\n   в”ңв”Җ Buzzer: Silent\n   в””в”Җ вҶ’ Next: Back to STATE_READY (waiting for PIR)\n```\n\n## Sensor & Hardware Timing\n\n| Component | Event | Timing | Action |\n|-----------|-------|--------|--------|\n| PIR Sensor | Motion detected | Immediate | Entry count, transition to CUSTOMER_DETECTED |\n| 3kg Scale | Product placed | ~1-2 seconds | Identify product, read weight |\n| 3kg Scale | Product removed | ~1 second | Ask for payment |\n| 1kg Scale | Coins inserted | ~1-2 seconds | Start coin accumulation |\n| 1kg Scale | Coins settling | 1000ms | Stabilize reading |\n| Payment Check | After settling | Immediate | Verify вүҘ required weight |\n| Success | 3 seconds | Auto-reset to READY |\n| Failure | 5 seconds | Timeout (or retry with more coins) |\n\n## LCD Display Messages\n\n```\nLINE 1-2 (20 chars max):\n- \"SmartPay Ready\"\n- \"Customer Entered\"\n- \"Product One\\nPHP5\"\n- \"Product Two\\nPHP10\"\n- \"Pay\\nProduct One\"\n- \"Pay\\nProduct Two\"\n- \"Payment OK\\nThank you!\"\n- \"Add More\\nCoins\"\n```\n\n## Serial Output Examples\n\n### Successful PHP5 Purchase\n```\nSmartPay Ready\nEntry: 1\nCustomer Entered\nProduct Removed. Pay Product One (PHP5).\nPay Product One (PHP5)\nCoins: 5.2g - OK\nPayment OK\nCustomer Left\nSmartPay Ready\n```\n\n### Failed PHP10, Then Retry Success\n```\nSmartPay Ready\nEntry: 2\nCustomer Entered\nProduct Removed. Pay Product Two (PHP10).\nPay Product Two (PHP10)\nCoins: 1.5g - INSUFFICIENT\nAdd More Coins\nCoins: 10.1g - OK\nPayment OK\nCustomer Left\nSmartPay Ready\n```\n\n## Dashboard Integration\n\nEach line sent to Serial is parsed by the dashboard:\n\n1. Parsed into an event (Entry, Product Removed, Coins OK, etc.)\n2. Stored in transaction log (LocalStorage + API)\n3. Updates stats (revenue, success rate, entry count, hourly breakdown)\n4. Updates LCD simulator on dashboard\n5. Transitions state machine (READY вҶ’ ENTERED вҶ’ PAY вҶ’ OK вҶ’ READY)\n\n## Debugging Checklist\n\n- [ ] Both scales calibrated and tested\n- [ ] PIR sensor responding to motion\n- [ ] LCD displaying status messages (I2C address correct?)\n- [ ] Buzzer beeping on events\n- [ ] Product weight thresholds set correctly\n- [ ] Coin weight thresholds set (PHP5 вүҘ5.0g, PHP10 вүҘ10.0g)\n- [ ] Serial output visible at 9600 baud\n- [ ] Dashboard connects via Web Serial\n- [ ] Demo flow runs end-to-end on dashboard\n
